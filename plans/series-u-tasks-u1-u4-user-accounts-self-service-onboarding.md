## 1. Task U1 — User Account Provisioning Linked to Employee

**Goal:** HR can provision a system login from the employee detail page (single or bulk), reset it, and deactivate it. Bidirectional FK between `employees` and `users`. Welcome email with temporary password.

**Permissions (seed update)**

Carries `tempPassword` and renders [`resources/views/emails/welcome.blade.php`](../api/resources/views/emails/welcome.blade.php) with: greeting, login URL (`config('app.frontend_url')`), email, temp password, "must change on first login" notice. Plain-text alt also generated by Laravel.

**Service**

[`api/app/Modules/HR/Services/UserProvisioningService.php`](../api/app/Modules/HR/Services/UserProvisioningService.php) — exact methods per spec:

- `provisionForEmployee(Employee $employee, array $options = []): User`
  - Wrapped in `DB::transaction`.
  - Throws `\DomainException` if `$employee->user_id` already set.
  - Generates email via private `generateEmail()` (sanitized first.last + numeric suffix collision handling) when not provided.
  - Generates 12-char random temp password (`Str::password(12, true, true, true, false)`).
  - Creates `User` with `must_change_password = true`, `is_active = true`, `password_changed_at = null`, `role_id` from `$options['role_id']` or default Employee role lookup by slug.
  - Stores hash in `password_history`.
  - Updates `$employee->user_id`.
  - Notifies user with `WelcomeNotification($tempPassword)` (skipped when `$options['send_welcome'] === false`).
  - Returns `$user->fresh(['role'])`.
- `deactivateForEmployee(Employee $employee): void` — flips `is_active`, deletes the user's `sessions` rows, flushes permissions cache.
- `resetPasswordForEmployee(Employee $employee): string` — generates new temp password, sets `must_change_password = true`, `failed_login_attempts = 0`, `locked_until = null`, persists to `password_history`, sends a "PasswordResetNotification" email; returns the temp password (not stored anywhere readable).
- `accountStatusForEmployee(Employee $employee): array` — `account_exists`, `is_active`, `is_locked`, `email`, `role`, `last_login_at`, `must_change_password`.
- `bulkProvision(array $employeeIds): array` — iterates, returns per-employee result `{employee_id, status: success|skipped|failed, message}`. Sub-transactions per employee so one failure does not roll all back.

## 4. Task U4 — Employee Onboarding Workflow

**Goal:** New employee = guided 8-step workflow. System auto-completes what it can; HR sees the rest as actionable buttons.

### 4.1 Backend

| column                    | type                                                       |

| employee_id               | `foreignId.unique.constrained.cascadeOnDelete`             |
| profile_completed_at      | `timestamp.nullable` (auto on employee create)             |
| shift_assigned_at         | `timestamp.nullable`                                       |
| leave_balances_initialized_at | `timestamp.nullable` (auto)                            |
| account_provisioned_at    | `timestamp.nullable`                                       |

| gov_ids_recorded_at       | `timestamp.nullable`                                       |
| banking_recorded_at       | `timestamp.nullable`                                       |
| completed_at              | `timestamp.nullable`                                       |
| reminder_sent_at          | `timestamp.nullable`                                       |
| timestamps                |                                                            |

**Service**

[`HR\Services\OnboardingService`](../api/app/Modules/HR/Services/OnboardingService.php):

- `initialize(Employee $employee): EmployeeOnboarding` — called from `EmployeeService::create()`. Runs in same `DB::transaction`. Marks `profile_completed_at`. Creates leave balances (calls `LeaveBalanceInitializer` from existing leave module if present; otherwise iterates active `LeaveType` rows, pro-rated by `(12 - hire_month) / 12`, writes `EmployeeLeaveBalance` rows). Marks `leave_balances_initialized_at`. Returns the onboarding record.
- `markStep(Employee, string $step): void` — `$step` ∈ enum.
- `recompute(Employee): EmployeeOnboarding` — recalc from real data: shift assigned (`employee_shift_assignments` exists), gov_ids_recorded (sss_no, philhealth_no, pagibig_no, tin all present), banking_recorded (bank_name & bank_account_no present), account_provisioned (`user_id` not null). This is the source-of-truth so the badge is always accurate even if HR edits via the regular form.
- `status(Employee): array` — returns ordered step list with `{key, label, completed_at|null, action_url|null}` for SPA consumption.
- `sendRemindersForStaleOnboardings(): int` — scheduled job: any onboarding open > 3 days without `completed_at` -> notify HR officers, set `reminder_sent_at`.

**Listener** — wire `EmployeeService::create()` to call `OnboardingService::initialize()`. After `UserProvisioningService::provisionForEmployee()` succeeds, call `OnboardingService::markStep('account_provisioned')`. Same for Shift assignment controller, Banking save (employee update with non-null bank fields), Gov-IDs save.

**Schedule** — register `sendRemindersForStaleOnboardings` daily at 09:00 in [`api/app/Console/Kernel.php`](../api/app/Console/Kernel.php) (or `routes/console.php`, depending on Laravel 11 setup — confirm during implementation).

**API**

- `GET /api/v1/employees/{employee}/onboarding` — `OnboardingResource`.
- `POST /api/v1/employees/{employee}/onboarding/recompute` — re-derives status (admin-only diagnostic).

**Permissions** — reuses `hr.employees.view` / `hr.employees.edit`. No new permissions.

**Tests** — [`tests/Feature/HR/OnboardingTest.php`](../api/tests/Feature/HR/OnboardingTest.php): creating an employee initializes onboarding + leave balances; provisioning account marks step; reminder job only notifies after 3 days.

### 4.2 Frontend
