# Track 2, Task 2: Edge WO Output Ingest (PLC) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` syntax.

**Goal:** Add `POST /edge/v1/output` so a PLC counter device (token ability `edge:output`) bound to a specific machine can push shot/cycle counts. The endpoint auto-resolves the active work order running on that machine, then reuses the battle-tested `WorkOrderOutputService::record()` to persist the output — meaning OEE, mold shot count, scrap rate, dashboard events, and idempotency all "just work."

**Architecture:**

```
PLC device (machine bound) → POST /edge/v1/output { good_count, reject_count?, defects?, idempotency_key }
  ↓
EdgeOutputController (ability:edge:output)
  ↓
EdgeOutputIngestService::ingest(EdgeDevice $device, array $payload, ?string $idemKey)
  ├─ requires device.machine_id set      → 422 if not provisioned
  ├─ finds the active WO on that machine → 422 if none (status=in_progress)
  ├─ resolves an "edge ingest" system user (cached) for recorded_by audit
  └─ delegates to WorkOrderOutputService::record($wo, $payload, $systemUserId, $idemKey)
  ↓
Standard WO output flow: outputs row + defects + WO totals + mold shots + event.
```

Key constraints reused from `WorkOrderOutputService`:
- WO must be `in_progress` (service already enforces).
- `good + reject > 0` (service already enforces).
- If defects supplied, sum must equal `reject_count` (service already enforces).
- Cumulative output cannot exceed `quantity_target` (service already enforces).
- Idempotency key cached 24h — duplicate replays return cached row.

New `machine_id` column on `edge_devices` (nullable FK to `machines`) — only PLC/IoT devices need it, kept nullable so scanner/caliper devices don't.

A dedicated system user `edge-system@ogami.internal` records the audit trail (created lazily and cached). This is so `WorkOrderOutput.recorded_by` (NOT NULL) is satisfied without overloading admin accounts.

**Tech Stack:** Laravel 11, Sanctum bearer tokens, Postgres. No SPA work.

## Background

- `WorkOrderOutputService::record(WorkOrder $wo, array $data, int $recordedBy, ?string $idemKey): WorkOrderOutput` — the canonical entry point. Already handles idempotency, locking, mold increments, and event dispatch. **DO NOT REINVENT.**
- `WorkOrder.machine_id` exists (mig 0079-ish). A machine can host at most one `in_progress` WO at a time in current data model — we still defensively pick the most recently `actual_start`-stamped one if multiples exist (treat as warning in response).
- `WorkOrder.status` is the `WorkOrderStatus` enum — `InProgress = 'in_progress'`.
- `EdgeDevice` exists (T2.0). Currently has no machine FK — add it via migration so PLC/IoT can target a machine.
- `Authenticatable` system user is needed because `WorkOrderOutput.recorded_by` is a FK to `users.id`. Lazily create `edge-system@ogami.internal` (role: lowest-priv, e.g. `employee`) on first ingest, cache its id.
- Sanctum's `ability:edge:output` alias already wired (T2.1 registered the alias). Reuse.
- HashIDs: inbound `machine_id` for admin endpoints comes as hash; decode via `app('hashids')->decode($v)[0] ?? null`. Edge endpoint itself does NOT take machine_id (device owns the binding).
- Audit log: `WorkOrderOutputRecorded` event already dispatches; we don't need to add anything for chain-of-custody beyond that.

## Payload Shape

```jsonc
// POST /api/v1/edge/v1/output
// Bearer: <PLC token with ability edge:output>
{
  "good_count": 12,           // required, int >= 0
  "reject_count": 0,          // optional, int >= 0, default 0
  "shift": "AM",              // optional, free string
  "remarks": "auto-PLC",      // optional
  "defects": [                // optional; if reject_count>0 and defects given, sum must equal reject_count
    { "defect_type_id": "qZ8nKx", "count": 0 }  // hash id; service expects raw int — controller decodes
  ],
  "idempotency_key": "press3-2026-06-14T22:15:03Z-001" // optional — recommended. PLC firmware sends this.
}
```

Response 201:

```jsonc
{
  "data": {
    "output_id":    "wK5mPx",
    "wo_id":        "qP4nZx",
    "wo_number":    "WO-202604-0007",
    "batch_code":   "WO-202604-0007-B03",
    "good_count":   12,
    "reject_count": 0,
    "recorded_at":  "2026-06-14T22:15:04+00:00"
  }
}
```

Errors:
- 422 `"device_not_bound_to_machine"` — `edge_devices.machine_id IS NULL`.
- 422 `"no_active_work_order"` — no in-progress WO on this machine.
- 422 plain (service-level) — exceed target, mismatched defects, no count, WO not in progress (replayed from `RuntimeException::getMessage()`).
- 403 — token lacks `edge:output`.
- 401 — no token.

### Task D: Edge output ingest service

**Files:** Create `api/app/Modules/Edge/Services/EdgeOutputIngestService.php`

- [ ] Step 1:

```php
<?php

declare(strict_types=1);

namespace App\Modules\Edge\Services;

use App\Modules\Auth\Models\Role;
use App\Modules\Auth\Models\User;
use App\Modules\Edge\Models\EdgeDevice;
use App\Modules\Production\Enums\WorkOrderStatus;
use App\Modules\Production\Models\WorkOrder;
use App\Modules\Production\Models\WorkOrderOutput;
use App\Modules\Production\Services\WorkOrderOutputService;
use Illuminate\Support\Facades\Cache;
use Illuminate\Validation\ValidationException;

/**
 * T2.2 — Edge → WO output ingest.
 *
 * Resolves the active WO running on the PLC device's bound machine and
 * delegates to WorkOrderOutputService::record() so idempotency, mold shot
 * tracking, scrap rate, and dashboard events all work identically to the
 * manual SPA path.
 */
class EdgeOutputIngestService
{
    private const SYSTEM_USER_EMAIL  = 'edge-system@ogami.internal';
    private const SYSTEM_USER_CACHE  = 'edge:system_user_id';

    public function __construct(private readonly WorkOrderOutputService $outputs) {}

    public function ingest(EdgeDevice $device, array $payload, ?string $idemKey = null): WorkOrderOutput
    {
        if (! $device->machine_id) {
            throw ValidationException::withMessages([
                'device' => ['device_not_bound_to_machine'],
            ]);
        }

        $wo = WorkOrder::query()
            ->where('machine_id', $device->machine_id)
            ->where('status', WorkOrderStatus::InProgress->value)
            ->orderByDesc('actual_start')
            ->first();

        if (! $wo) {
            throw ValidationException::withMessages([
                'machine' => ['no_active_work_order'],
            ]);
        }

        return $this->outputs->record($wo, $payload, $this->systemUserId(), $idemKey);
    }

    /**
     * Lazily provision and cache the audit-trail user the edge stack uses
     * for recorded_by on output rows. Role is `employee` (lowest priv).
     */
    private function systemUserId(): int
    {
        $cached = Cache::get(self::SYSTEM_USER_CACHE);
        if (is_int($cached)) return $cached;

        $user = User::query()->where('email', self::SYSTEM_USER_EMAIL)->first();
        if (! $user) {
            $roleId = Role::query()->where('slug', 'employee')->value('id')
                ?? Role::query()->orderBy('id')->value('id');
            $user = User::create([
                'name'      => 'Edge System',
                'email'     => self::SYSTEM_USER_EMAIL,
                'password'  => bcrypt(\Illuminate\Support\Str::random(40)),
                'role_id'   => $roleId,
                'is_active' => true,
            ]);
        }
        Cache::forever(self::SYSTEM_USER_CACHE, $user->id);
        return $user->id;
    }
}
```

NOTE: If `User::create` requires additional non-nullable columns (e.g. `employee_id`), the implementer adapts — read the users migration first. If `employee_id` is non-null, skip the create path entirely and require seeding the user separately (then make the cache lookup throw a clear error if missing).

### Task F: Controller

**Files:** Create `api/app/Modules/Edge/Controllers/EdgeOutputController.php`

- [ ] Step 1:

```php
<?php

declare(strict_types=1);

namespace App\Modules\Edge\Controllers;

use App\Modules\Edge\Models\EdgeDevice;
use App\Modules\Edge\Requests\OutputIngestRequest;
use App\Modules\Edge\Services\EdgeOutputIngestService;
use Illuminate\Http\JsonResponse;

class EdgeOutputController
{
    public function __construct(private readonly EdgeOutputIngestService $service) {}

    public function ingest(OutputIngestRequest $request): JsonResponse
    {
        /** @var EdgeDevice $device */
        $device = $request->user();

        $payload = $request->validated();
        // Decode hash-ID defect_type_ids → ints for the underlying WO output service.
        if (! empty($payload['defects'])) {
            $payload['defects'] = array_map(function (array $d) {
                $decoded = app('hashids')->decode((string) ($d['defect_type_id'] ?? ''));
                return [
                    'defect_type_id' => empty($decoded) ? 0 : (int) $decoded[0],
                    'count'          => (int) ($d['count'] ?? 0),
                ];
            }, $payload['defects']);
        }

        $idem = $payload['idempotency_key'] ?? $request->header('X-Idempotency-Key');
        unset($payload['idempotency_key']);

        $output = $this->service->ingest($device, $payload, $idem);

        return response()->json([
            'data' => [
                'output_id'    => $output->hash_id,
                'wo_id'        => $output->workOrder?->hash_id,
                'wo_number'    => $output->workOrder?->wo_number,
                'batch_code'   => $output->batch_code,
                'good_count'   => (int) $output->good_count,
                'reject_count' => (int) $output->reject_count,
                'recorded_at'  => optional($output->recorded_at)->toIso8601String(),
            ],
        ], 201);
    }
}
```

NOTE: `WorkOrderOutput` needs a `workOrder()` relationship for the `->workOrder?->hash_id` access to work. Check `api/app/Modules/Production/Models/WorkOrderOutput.php` — if missing, fall back to a fresh lookup `WorkOrder::find($output->work_order_id)?->hash_id`.

use Tests\TestCase;

class EdgeOutputIngestTest extends TestCase
{
    use RefreshDatabase;

    {
        $wo = WorkOrder::factory()->create([
            'machine_id'        => $m->id,

    {
        return $this->withHeaders(array_merge([
