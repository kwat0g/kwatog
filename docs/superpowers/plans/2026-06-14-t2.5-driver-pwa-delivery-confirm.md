# Track 2, Task 5: Driver PWA Shell + Delivery Confirm — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` syntax.

**Goal:** Give drivers a phone-friendly Progressive Web App so they can see their own delivery assignments, update status as they progress (`scheduled` → `loading` → `in_transit` → `delivered`), and capture a receipt photo on the spot. The driver does NOT confirm — confirmation (which auto-creates the invoice) remains with the CRM officer. The PWA is a thin SPA layer on top of new driver-scoped backend routes that wrap the existing `DeliveryService`.

**Architecture:**

```
Backend                                              Frontend
───────                                              ────────
GET  /api/v1/driver/deliveries        ─── DriverDeliveryController (scoped to driver_id = auth user)
GET  /api/v1/driver/deliveries/{hid}                            │
PATCH /api/v1/driver/deliveries/{hid}/status                    ▼
POST /api/v1/driver/deliveries/{hid}/receipt          DriverLayout (no sidebar, mobile-first)
                                                       /driver           DriverDeliveryList
                                                       /driver/:hid      DriverDeliveryDetail
                                                       /driver/:hid/photo DriverPhotoCapture
```

Backend reuses `DeliveryService::updateStatus()` and `DeliveryService::uploadReceiptPhoto()`. Driver scope enforced by ownership filter (`where('driver_id', auth()->id())`) in the new service method. Driver can transition only:

- `scheduled → loading`
- `loading → in_transit`
- `in_transit → delivered`

`delivered → confirmed` is gated by `supply_chain.deliveries.create` permission (existing CRM officer flow). Driver attempts to confirm → 403.

**PWA shape:** minimum-viable manifest + theme/meta tags so iOS + Android "Add to Home Screen" produces an app-like icon and standalone window. NO service worker / offline cache — drivers need fresh delivery state, and offline queueing would need IndexedDB persistence layer that's out of scope. This is the same constraint Cubic / Fairmatic apply for the thesis demo.

**Tech Stack:** Laravel 11 backend, React 18 SPA, Sanctum SPA cookie session reused (driver logs in via existing `/login`).

## Background

- `Delivery.driver_id` is `FK → users.id`, nullable. The driver "logs in" with their normal user account; ownership is by FK, not role.
- `DeliveryStatus` enum: `Scheduled | Loading | InTransit | Delivered | Confirmed | Cancelled`.
- `DeliveryService::updateStatus(Delivery $d, DeliveryStatus $next, ?string $note = null): Delivery` — enforces forward-only transitions; stamps `actual_departure_at` on `in_transit` and `actual_arrival_at` on `delivered`.
- `DeliveryService::uploadReceiptPhoto(Delivery $d, UploadedFile $file, ?User $by = null): Delivery` — used for receipt photo upload after delivery (writes `delivery_proofs` row).
- Existing controller `DeliveryController::updateStatus / uploadReceipt` are permission-gated for CRM/dispatch. We do NOT change those — new driver routes are a parallel, narrower surface.
- The seeded `driver` role exists only so HR can tag user accounts as drivers in their profile/admin view. Routes themselves do not check role; they check `driver_id` ownership.
- iOS Safari requires `apple-mobile-web-app-capable=yes` + `apple-touch-icon` for "Add to Home Screen" to launch standalone. Android picks it up from the manifest.
- No service worker because offline delivery updates would need IndexedDB queue + sync — significant additional surface and not required for the demo.

```php
<?php

declare(strict_types=1);

namespace App\Modules\SupplyChain\Services;

use App\Modules\Auth\Models\User;
use App\Modules\SupplyChain\Enums\DeliveryStatus;
use App\Modules\SupplyChain\Models\Delivery;
use Illuminate\Contracts\Pagination\LengthAwarePaginator;
use Illuminate\Http\UploadedFile;
use Illuminate\Validation\ValidationException;

/**
 * T2.5 — Self-scoped delivery surface for drivers.
 *
 * All reads/writes are filtered to deliveries whose `driver_id` equals
 * the authenticated user. Status transitions and receipt-photo uploads
 * delegate to the existing DeliveryService. Confirmation (which auto-
 * creates an invoice) is intentionally NOT exposed here — drivers
 * cannot confirm; that's the CRM officer's job.
 */
class DriverDeliveryService
{
    /** Driver-allowed forward transitions. */
    private const ALLOWED = [
        'scheduled'  => 'loading',
        'loading'    => 'in_transit',
        'in_transit' => 'delivered',
    ];

    public function __construct(private readonly DeliveryService $deliveries) {}

    public function list(User $driver, array $filters): LengthAwarePaginator
    {
        $q = Delivery::query()
            ->where('driver_id', $driver->id)
            ->with([
                'salesOrder:id,so_number,customer_id',
                'salesOrder.customer:id,name',
                'vehicle:id,plate_number',
            ]);

        if (! empty($filters['status'])) {
            $q->where('status', $filters['status']);
        } else {
            // Default: hide finalised so the driver sees actionable rows.
            $q->whereNotIn('status', [DeliveryStatus::Confirmed->value, DeliveryStatus::Cancelled->value]);
        }

        return $q->orderBy('planned_departure_at')
            ->paginate(min((int) ($filters['per_page'] ?? 25), 100));
    }

    public function show(User $driver, Delivery $delivery): Delivery
    {
        $this->assertOwned($driver, $delivery);
        return $this->deliveries->show($delivery);
    }

    public function updateStatus(User $driver, Delivery $delivery, string $next): Delivery
    {
        $this->assertOwned($driver, $delivery);

        $current = $delivery->status->value;
        $allowed = self::ALLOWED[$current] ?? null;
        if ($allowed !== $next) {
            throw ValidationException::withMessages([
                'status' => ["driver_cannot_transition_from_{$current}_to_{$next}"],
            ]);
        }

        return $this->deliveries->updateStatus($delivery, DeliveryStatus::from($next), null);
    }

    public function uploadReceipt(User $driver, Delivery $delivery, UploadedFile $file): Delivery
    {
        $this->assertOwned($driver, $delivery);
        return $this->deliveries->uploadReceiptPhoto($delivery, $file, $driver);
    }

    private function assertOwned(User $driver, Delivery $delivery): void
    {
        if ($delivery->driver_id !== $driver->id) {
            abort(404); // 404 hides existence — narrower than 403.
        }
    }
}
```

NOTE: `DeliveryService::show()` returns the delivery with the relations the SPA needs. If the eager loads differ from what the driver detail page needs, override `show()` here to load a leaner set — `items.product`, `vehicle`, `salesOrder.customer`, `proofs`.

namespace App\Modules\SupplyChain\Controllers;

class DriverDeliveryController
{
    public function __construct(private readonly DriverDeliveryService $service) {}

    public function index(Request $request): AnonymousResourceCollection
    {
        return DeliveryResource::collection(
            $this->service->list($request->user(), $request->all()),
        );
    }

    public function show(Request $request, Delivery $delivery): DeliveryResource
    {
        return new DeliveryResource(
            $this->service->show($request->user(), $delivery),
        );
    }

    public function updateStatus(DriverUpdateStatusRequest $request, Delivery $delivery): DeliveryResource
    {
        return new DeliveryResource(
            $this->service->updateStatus(
                $request->user(),
                $delivery,
                (string) $request->validated('status'),
            ),
        );
    }

    public function uploadReceipt(DriverUploadReceiptRequest $request, Delivery $delivery): DeliveryResource
    {
        return new DeliveryResource(
            $this->service->uploadReceipt(
                $request->user(),
                $delivery,
                $request->file('photo'),
            ),
        );
    }
}

            'so_number'    => 'SO-T-'.substr(uniqid(), -5),
            'customer_id'  => $customer->id,

            'planned_departure_at' => now(),
            'planned_arrival_at'   => now()->addHours(2),

import { client } from './client';
import type { DriverDelivery, DriverPaginated } from '@/types/driver';

export const driverApi = {
  listDeliveries: (params?: Record<string, string>) =>
    client.get<DriverPaginated<DriverDelivery>>('/driver/deliveries', { params }).then(r => r.data),
  showDelivery: (id: string) =>
    client.get<{ data: DriverDelivery }>(`/driver/deliveries/${id}`).then(r => r.data.data),
  updateStatus: (id: string, status: DriverDelivery['status']) =>
    client.patch<{ data: DriverDelivery }>(`/driver/deliveries/${id}/status`, { status }).then(r => r.data.data),

    return client.post<{ data: DriverDelivery }>(`/driver/deliveries/${id}/receipt`, form, {
      headers: { 'Content-Type': 'multipart/form-data' },
    }).then(r => r.data.data);

import { useMutation, useQueryClient } from '@tanstack/react-query';
import { useRef, useState } from 'react';
import { Link, useNavigate, useParams } from 'react-router-dom';
import { toast } from 'sonner';
import { driverApi } from '@/api/driver';

export default function DriverPhotoCapture() {
  const { id = '' } = useParams();
  const navigate = useNavigate();
  const qc = useQueryClient();
  const fileRef = useRef<HTMLInputElement>(null);
  const [preview, setPreview] = useState<string | null>(null);
  const [file, setFile] = useState<File | null>(null);

  const upload = useMutation({
    mutationFn: () => {
      if (!file) throw new Error('no file');
      return driverApi.uploadReceipt(id, file);
    },
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: ['driver'] });
      toast.success('Receipt uploaded.');
      navigate(`/driver/${id}`);
    },
    onError: () => toast.error('Upload failed.'),
  });

  return (
    <div className="space-y-4">
      <Link to={`/driver/${id}`} className="text-sm text-zinc-500">← Back to delivery</Link>
      <h1 className="text-lg font-semibold">Receipt Photo</h1>

      <input
        ref={fileRef}
        type="file"
        accept="image/*"
        capture="environment"
        className="hidden"
        onChange={(e) => {
          const f = e.target.files?.[0];
          if (!f) return;
          setFile(f);
          setPreview(URL.createObjectURL(f));
        }}
      />

      {preview ? (
        <img src={preview} alt="receipt" className="w-full rounded-lg" />
      ) : (
        <div className="aspect-[4/3] rounded-lg border-2 border-dashed border-zinc-300 dark:border-zinc-700 flex items-center justify-center text-zinc-500">

      >
        {preview ? 'Retake' : 'Open Camera'}
      </button>

      <button
        type="button"
        disabled={!file || upload.isPending}
        onClick={() => upload.mutate()}
        className="w-full rounded-lg bg-indigo-600 text-white py-3 font-medium disabled:opacity-60"
