# Track 1, Task 1: Auto-Detect Overtime from Punches — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When biometric punches reveal an employee worked past `shift_end + threshold`, auto-create a draft `OvertimeRequest` so the supervisor can 1-tap approve instead of the employee filing a manual request.

**Architecture:** Add an idempotent `OvertimeService::autoDetectFromAttendance()` method invoked from `AttendanceService::create/update` and `DTRImportService::import` (after each row's `Attendance` save). The detection runs on saved Attendance rows, computes extra minutes beyond shift end, and inserts an OT row when (a) extras exceed the configured threshold, (b) no OT row already exists for that employee+date, and (c) the feature flag is on. Pure-function detection logic lives on the service so it is unit-testable without DB. A new `is_auto_detected` boolean on `overtime_requests` lets the UI badge auto-rows and lets future code distinguish manual vs detected.

**Tech Stack:** Laravel 11 (PHP 8.3), PHPUnit/Pest, Postgres 16. Reuse existing `SettingsService` for feature flag + threshold. Reuse existing `OvertimeRequestSubmitted` event so the supervisor notification path already in place fires automatically.

## Detection Math (the rule)

Given an `Attendance` row that has both `time_in` and `time_out`, with the `shift` resolved:

```
shift_end_anchor = parse(date + shift.end_time)            # if night shift and end <= start, add 1 day
extra_minutes    = max(0, time_out - shift_end_anchor in minutes)
```

If `extra_minutes >= threshold_minutes` AND no existing OT row for (employee_id, date) → create one with:
- `hours_requested = round(extra_minutes / 60, 1)` (matches the `decimal:1` cast)
- `reason = "Auto-detected from biometric punch (worked {extra_minutes} minutes past shift end)"`
- `status = pending`
- `is_auto_detected = true`

Skip when:
- `time_out` is null
- the day is a rest day or holiday AND `regular_hours == 0` (worker just punched in by mistake — let the human file)
- existing OT row exists for that employee+date in any status (idempotency guard)
- feature flag is off
- `extra_minutes < threshold`

```php
/**
 * Pure helper — minutes the punch-out is past the resolved shift end.
 * Clamps below zero to 0 so the caller never persists negative OT.
 */
public function extraMinutesPastShiftEnd(string $shiftEnd, string $timeOut): int
{
    $end = \Carbon\CarbonImmutable::parse($shiftEnd);
    $out = \Carbon\CarbonImmutable::parse($timeOut);
    if ($out->lessThanOrEqualTo($end)) {
        return 0;
    }
    return (int) $end->diffInMinutes($out);
}
```

After `extraMinutesPastShiftEnd`, add:

```php
/**
 * Idempotent: scans an Attendance row and creates a draft OT request
 * if punches show work past shift end beyond the configured threshold.
 *
 * Returns the created OT row, or null if any guard short-circuited.
 */
public function autoDetectFromAttendance(\App\Modules\Attendance\Models\Attendance $a): ?OvertimeRequest
{
    $settings = app(\App\Common\Services\SettingsService::class);
    if (! (bool) $settings->get('attendance.auto_ot_detect.enabled', true)) {
        return null;
    }
    $threshold = (int) $settings->get('attendance.auto_ot_detect.threshold_minutes', 30);

    if (! $a->time_in || ! $a->time_out) {
        return null;
    }

    $a->loadMissing('shift');
    $shift = $a->shift;
    if (! $shift) {
        return null; // no shift → cannot anchor end time
    }

    // Build the shift-end anchor on the attendance date.
    $date = \Carbon\CarbonImmutable::parse((string) $a->date);
    $endTime = $shift->end_time instanceof \Carbon\CarbonInterface
        ? $shift->end_time->format('H:i:s')
        : (string) $shift->end_time;
    $startTime = $shift->start_time instanceof \Carbon\CarbonInterface
        ? $shift->start_time->format('H:i:s')
        : (string) $shift->start_time;

    $shiftEnd = \Carbon\CarbonImmutable::parse($date->toDateString() . ' ' . $endTime);
    $shiftStart = \Carbon\CarbonImmutable::parse($date->toDateString() . ' ' . $startTime);
    if ($shiftEnd->lessThanOrEqualTo($shiftStart)) {
        $shiftEnd = $shiftEnd->addDay();
    }

    $extra = $this->extraMinutesPastShiftEnd(
        $shiftEnd->toDateTimeString(),
        \Carbon\CarbonImmutable::parse($a->time_out)->toDateTimeString(),
    );
    if ($extra < $threshold) {
        return null;
    }

    // Idempotency: skip if any OT row exists for (employee, date).
    $exists = OvertimeRequest::query()
        ->where('employee_id', $a->employee_id)
        ->whereDate('date', $a->date)
        ->exists();
    if ($exists) {
        return null;
    }

    $hours = round($extra / 60, 1);

    return \DB::transaction(function () use ($a, $hours, $extra) {
        $ot = OvertimeRequest::create([
            'employee_id'      => $a->employee_id,
            'date'             => $a->date,
            'hours_requested'  => $hours,
            'reason'           => "Auto-detected from biometric punch (worked {$extra} minutes past shift end).",
            'is_auto_detected' => true,
        ]);
        // Default DB status is 'pending' — no need to set explicitly.
        $ot->load('employee');
        event(new \App\Modules\Attendance\Events\OvertimeRequestSubmitted($ot));
        return $ot;
    });
}
```
