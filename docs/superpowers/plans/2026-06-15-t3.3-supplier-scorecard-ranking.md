# T3.3 — Supplier Scorecard Ranking + Tiering + Deterioration Alert Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Polish the existing supplier scorecard surface — already has `SupplierPerformanceService::compute()`, monthly snapshots, the `purchasing:recompute-supplier-performance` cron and the `GET /vendors/{vendor}/performance` endpoint. T3.3 adds three additive gaps: A/B/C/D tier letter on every snapshot, a ranking endpoint that lists vendors by score for a period, and a queued listener that alerts purchasing officers when a vendor's score drops ≥ 20 points month-over-month.

**Architecture:** ONE additive migration (0189) adds a nullable `tier varchar(1)` column to `supplier_performance_snapshots`. `SupplierPerformanceService::compute()` is extended to write the tier and to dispatch a new `SupplierPerformanceComputed` event afterCommit. `AlertOnSupplierDeterioration` (queued, swallows all `\Throwable`) compares the new snapshot's `overall_score` against the prior month's snapshot for the same vendor and notifies users with role `purchasing_officer` when the drop is ≥ 20 points. The controller gets a `ranking()` method serving `GET /api/v1/purchasing/vendors/ranking`.

**Tech Stack:** Laravel 11, PostgreSQL 16, existing `NotificationService::send`, existing `purchasing.suppliers.performance.view` permission. Event/listener wiring follows the established T3.1 pattern in `AppServiceProvider::boot()` (`Event::listen(...)`). No new packages, no SPA work.

```php
/**
 * GET /api/v1/purchasing/vendors/ranking
 *
 * Query: period_year, period_month, tier (A|B|C|D), limit (1..100, default 50).
 * Default period = previous calendar month.
 */
public function ranking(Request $request): JsonResponse
{
    $validated = $request->validate([
        'period_year'  => ['nullable', 'integer', 'min:2000', 'max:2100'],
        'period_month' => ['nullable', 'integer', 'min:1', 'max:12'],
        'tier'         => ['nullable', 'string', \Illuminate\Validation\Rule::in(['A','B','C','D'])],
        'limit'        => ['nullable', 'integer', 'min:1', 'max:100'],
    ]);

    $default = Carbon::now()->subMonthNoOverflow()->startOfMonth();
    $year  = (int) ($validated['period_year']  ?? $default->year);
    $month = (int) ($validated['period_month'] ?? $default->month);
    $limit = (int) ($validated['limit'] ?? 50);

    $query = \App\Modules\Purchasing\Models\SupplierPerformanceSnapshot::query()
        ->with(['vendor:id,name'])
        ->where('period_year',  $year)
        ->where('period_month', $month)
        ->orderByRaw('overall_score DESC NULLS LAST')
        ->limit($limit);

    if (!empty($validated['tier'])) {
        $query->where('tier', $validated['tier']);
    }

    $rows = $query->get()
        ->filter(fn ($s) => $s->vendor !== null)
        ->sortBy(fn ($s) => $s->vendor->name)            // tie-break by name
        ->sortByDesc(fn ($s) => $s->overall_score ?? -1) // primary: score desc
        ->values();

    return response()->json([
        'data' => $rows->map(fn ($s) => [
            'vendor' => [
                'id'   => $s->vendor->hash_id,
                'name' => $s->vendor->name,
            ],
            'period_year'           => $s->period_year,
            'period_month'          => $s->period_month,
            'overall_score'         => $s->overall_score,
            'tier'                  => $s->tier,
            'on_time_delivery_rate' => $s->on_time_delivery_rate,
            'quality_pass_rate'     => $s->quality_pass_rate,
            'ncr_rate'              => $s->ncr_rate,
            'po_count'              => $s->po_count,
            'grn_count'             => $s->grn_count,
        ])->values(),
        'meta' => [
            'period_year'  => $year,
            'period_month' => $month,
            'count'        => $rows->count(),
        ],
    ]);
}
```

## Task 5 — T3.3.C: `SupplierPerformanceComputed` event + `AlertOnSupplierDeterioration` listener

**Behaviour:**
- Event dispatched `DB::afterCommit(...)` at the end of `SupplierPerformanceService::compute()`. Carries `SupplierPerformanceSnapshot $snapshot`.

`api/app/Modules/Purchasing/Listeners/AlertOnSupplierDeterioration.php`:

```php
<?php

declare(strict_types=1);

namespace App\Modules\Purchasing\Listeners;

use App\Common\Services\NotificationService;
use App\Modules\Auth\Models\User;
use App\Modules\Purchasing\Events\SupplierPerformanceComputed;
use App\Modules\Purchasing\Models\SupplierPerformanceSnapshot;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Support\Facades\Log;

/**
 * T3.3.C — When a vendor's overall_score drops by 20+ points vs the prior
 * month's snapshot, notify all active users with role 'purchasing_officer'.
 *
 * Queued; swallows all \Throwable so a failure here never blocks the
 * SupplierPerformanceService::compute() workflow that dispatched the event.
 */
class AlertOnSupplierDeterioration implements ShouldQueue
{
    public const DROP_THRESHOLD = 20.0;

    public function __construct(private readonly NotificationService $notifications) {}

    public function handle(SupplierPerformanceComputed $event): void
    {
        try {
            $current = $event->snapshot;
            if ($current->overall_score === null) return;

            $prior = $this->priorMonthSnapshot($current);
            if ($prior === null || $prior->overall_score === null) return;

            $drop = (float) $prior->overall_score - (float) $current->overall_score;
            if ($drop < self::DROP_THRESHOLD) return;

            $audience = User::query()
                ->whereHas('role', fn ($q) => $q->where('slug', 'purchasing_officer'))
                ->where('is_active', true)
                ->get();

            if ($audience->isEmpty()) return;

            $vendorName = $current->vendor?->name ?? "Vendor #{$current->vendor_id}";
            $period     = sprintf('%04d-%02d', $current->period_year, $current->period_month);

            $this->notifications->send($audience, 'purchasing.supplier_deterioration', [
                'title'       => "Supplier deterioration — {$vendorName}",
                'message'     => sprintf(
                    '%s overall score dropped %.1f → %.1f (-%.1f) for %s.',
                    $vendorName,
                    (float) $prior->overall_score,
                    (float) $current->overall_score,
                    $drop,
                    $period,
                ),
                'link_to'     => $current->vendor
                    ? "/purchasing/vendors/{$current->vendor->hash_id}/performance"
                    : null,
                'entity_type' => 'supplier_performance_snapshot',
                'entity_id'   => $current->hash_id,
            ]);
        } catch (\Throwable $e) {
            Log::warning('AlertOnSupplierDeterioration failed', [
                'snapshot_id' => $event->snapshot->id ?? null,
                'error'       => $e->getMessage(),
            ]);
        }
    }

    private function priorMonthSnapshot(SupplierPerformanceSnapshot $current): ?SupplierPerformanceSnapshot
    {
        $year  = $current->period_year;
        $month = $current->period_month - 1;
        if ($month < 1) {
            $month = 12;
            $year  = $year - 1;
        }

        return SupplierPerformanceSnapshot::query()
            ->where('vendor_id',    $current->vendor_id)
            ->where('period_year',  $year)
            ->where('period_month', $month)
            ->first();
    }
}
```
