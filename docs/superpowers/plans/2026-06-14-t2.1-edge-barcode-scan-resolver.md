# Track 2, Task 1: Edge Barcode Scan Resolver — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` syntax.

**Goal:** Add `POST /edge/v1/scan` so a factory-floor barcode scanner (token ability `edge:scan`) can POST a scanned string and get back a typed entity (`Item`, `PurchaseOrder`, `WorkOrder`, `GoodsReceiptNote`) plus a list of `suggested_actions` the device should offer to the operator (e.g. "Open GRN for this PO", "Report output on this WO"). The resolver makes the device dumb — it only POSTs the raw barcode and renders whatever the server tells it to.

**Architecture:** New `EdgeScanController::resolve()` calls `EdgeScanResolverService::resolve(string $barcode, ?array $context)`. Resolver normalises the barcode (`trim`+`strtoupper`) then dispatches by prefix:

```
WO-*    → WorkOrder      (lookup by wo_number)
PO-*    → PurchaseOrder  (lookup by po_number)
GRN-*   → GoodsReceiptNote (lookup by grn_number)
default → Item            (lookup by code)
```

Each branch returns:
- `type` — `'work_order' | 'purchase_order' | 'goods_receipt_note' | 'item' | 'unknown'`
- `entity` — minimal projection (hash_id, number, status, related labels)
- `suggested_actions` — array of `{ action, label, params }` the device can render as buttons

`suggested_actions` are state-aware (e.g. only suggest `report_output` if WO is `in_progress`; only suggest `open_grn` if PO is `approved`/`sent`/`partially_received`). Unknown barcodes return `type='unknown'` with empty actions (HTTP 200, never 404 — the scanner shows "Unrecognised barcode").

`context` (optional) carries device state — current WO id, current GRN id — so the resolver can prioritise actions (e.g. if scanner is bound to a WO and scans an Item, suggest `issue_to_wo`). All context IDs are hash_ids; resolver decodes them.

**Tech Stack:** Laravel 11, Sanctum bearer tokens (already wired in T2.0). No SPA work. Pure server-side resolver.

## Background

- T2.0 landed `edge_device` guard + `/edge/v1/health` endpoint. Tokens for `barcode_scanner` devices carry `abilities=['edge:scan']` (see `EdgeDeviceType::abilities()`).
- Sanctum exposes `Route::middleware('ability:edge:scan')` to gate by token ability — the existing alias works because `EdgeDevice` uses the `HasApiTokens` trait.
- Items have `code` (no separate `barcode` column today). Treat `code` as the barcode payload. PO `po_number`, WO `wo_number`, GRN `grn_number` are the canonical scannable IDs.
- Hash IDs are exposed via `$model->hash_id` (HasHashId trait). Inbound `context.*_id` values are hash IDs and need decoding via `app('hashids')->decode($v)[0] ?? null`.
- `currentAccessToken()->can('edge:scan')` is the ability check (used via middleware `ability:edge:scan` rather than inline).
- No localisation today — return English labels.
- The full URL is `/api/v1/edge/v1/scan` (same `/api/v1` + `/edge/v1` nesting as T2.0).

```php
<?php

declare(strict_types=1);

namespace App\Modules\Edge\Services;

use App\Modules\Inventory\Models\GoodsReceiptNote;
use App\Modules\Inventory\Models\Item;
use App\Modules\Production\Models\WorkOrder;
use App\Modules\Purchasing\Models\PurchaseOrder;

/**
 * T2.1 — Resolves a scanned barcode to a typed entity + state-aware
 * suggested actions. Dispatch by canonical prefix:
 *   WO-*  → work order, PO-*  → purchase order,
 *   GRN-* → goods receipt note, otherwise → item.code.
 *
 * Unknown barcodes return type='unknown' with no actions. The device
 * renders "Unrecognised barcode" on that response.
 */
class EdgeScanResolverService
{
    public function resolve(string $barcode, array $context = []): array
    {
        $code = strtoupper(trim($barcode));
        if ($code === '') {
            return $this->unknown();
        }

        // Prefix dispatch.
        return match (true) {
            str_starts_with($code, 'WO-')  => $this->resolveWorkOrder($code, $context),
            str_starts_with($code, 'PO-')  => $this->resolvePurchaseOrder($code, $context),
            str_starts_with($code, 'GRN-') => $this->resolveGrn($code, $context),
            default                        => $this->resolveItem($code, $context),
        };
    }

    private function resolveWorkOrder(string $code, array $context): array
    {
        $wo = WorkOrder::query()
            ->with(['product:id,name'])
            ->where('wo_number', $code)
            ->first();
        if (! $wo) return $this->unknown();

        $actions = [];
        if (in_array($wo->status, ['in_progress', 'released'], true)) {
            $actions[] = [
                'action' => 'report_output',
                'label'  => 'Report output',
                'params' => ['wo_id' => $wo->hash_id],
            ];
            $actions[] = [
                'action' => 'report_defect',
                'label'  => 'Report defect',
                'params' => ['wo_id' => $wo->hash_id],
            ];
        }
        $actions[] = [
            'action' => 'view_wo',
            'label'  => 'View WO',
            'params' => ['id' => $wo->hash_id],
        ];

        return [
            'type'    => 'work_order',
            'entity'  => [
                'id'                => $wo->hash_id,
                'wo_number'         => $wo->wo_number,
                'product'           => $wo->product?->name,
                'status'            => $wo->status,
                'quantity_target'   => (int) $wo->quantity_target,
                'quantity_produced' => (int) ($wo->quantity_produced ?? 0),
            ],
            'suggested_actions' => $actions,
        ];
    }

    private function resolvePurchaseOrder(string $code, array $context): array
    {
        $po = PurchaseOrder::query()
            ->with(['vendor:id,name'])
            ->where('po_number', $code)
            ->first();
        if (! $po) return $this->unknown();

        $actions = [];
        if (in_array($po->status, ['approved', 'sent', 'partially_received'], true)) {
            $actions[] = [
                'action' => 'open_grn',
                'label'  => "Receive against {$po->po_number}",
                'params' => ['po_id' => $po->hash_id],
            ];
        }
        $actions[] = [
            'action' => 'view_po',
            'label'  => 'View PO',
            'params' => ['id' => $po->hash_id],
        ];

        return [
            'type'    => 'purchase_order',
            'entity'  => [
                'id'        => $po->hash_id,
                'po_number' => $po->po_number,
                'vendor'    => $po->vendor?->name,
                'status'    => $po->status,
            ],
            'suggested_actions' => $actions,
        ];
    }

    private function resolveGrn(string $code, array $context): array
    {
        $grn = GoodsReceiptNote::query()
            ->where('grn_number', $code)
            ->first();
        if (! $grn) return $this->unknown();

        return [
            'type'    => 'goods_receipt_note',
            'entity'  => [
                'id'         => $grn->hash_id,
                'grn_number' => $grn->grn_number,
                'status'     => $grn->status,
            ],
            'suggested_actions' => [[
                'action' => 'view_grn',
                'label'  => 'View GRN',
                'params' => ['id' => $grn->hash_id],
            ]],
        ];
    }

    private function resolveItem(string $code, array $context): array
    {
        $item = Item::query()->where('code', $code)->first();
        if (! $item) return $this->unknown();

        $actions = [];

        // Context-aware: scanner bound to an active WO → suggest issue.
        $woHash = $context['wo_id'] ?? null;
        if (is_string($woHash) && $woHash !== '') {
            $woId = $this->decodeHashId($woHash);
            $wo = $woId ? WorkOrder::query()->find($woId) : null;
            if ($wo && in_array($wo->status, ['released', 'in_progress'], true)) {
                $actions[] = [
                    'action' => 'issue_to_wo',
                    'label'  => "Issue to {$wo->wo_number}",
                    'params' => ['item_id' => $item->hash_id, 'wo_id' => $wo->hash_id],
                ];
            }
        }

        // Context-aware: scanner bound to a draft GRN → suggest add line.
        $grnHash = $context['grn_id'] ?? null;
        if (is_string($grnHash) && $grnHash !== '') {
            $grnId = $this->decodeHashId($grnHash);
            $grn = $grnId ? GoodsReceiptNote::query()->find($grnId) : null;
            if ($grn && $grn->status === 'draft') {
                $actions[] = [
                    'action' => 'add_to_grn',
                    'label'  => "Add to {$grn->grn_number}",
                    'params' => ['item_id' => $item->hash_id, 'grn_id' => $grn->hash_id],
                ];
            }
        }

        $actions[] = [
            'action' => 'view_item',
            'label'  => 'View item',
            'params' => ['id' => $item->hash_id],
        ];

        return [
            'type'    => 'item',
            'entity'  => [
                'id'              => $item->hash_id,
                'code'            => $item->code,
                'name'            => $item->name,
                'item_type'       => $item->item_type?->value,
                'unit_of_measure' => $item->unit_of_measure,
            ],
            'suggested_actions' => $actions,
        ];
    }

    private function unknown(): array
    {
        return ['type' => 'unknown', 'entity' => null, 'suggested_actions' => []];
    }

    private function decodeHashId(string $hash): ?int
    {
        $decoded = app('hashids')->decode($hash);
        return empty($decoded) ? null : (int) $decoded[0];
    }
}
```

## Acceptance Criteria

1. `POST /api/v1/edge/v1/scan` resolves a `WO-*` barcode to the WorkOrder with state-aware actions.
2. Same endpoint resolves `PO-*` to PurchaseOrder, `GRN-*` to GoodsReceiptNote, otherwise to Item by `code`.
3. Item scans honour `context.wo_id` / `context.grn_id` to prepend `issue_to_wo` / `add_to_grn` actions.
