# Track 1, Task 2: Auto-Email Payslip PDF on Finalize — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When a payroll period is finalized, every employee with an email on file automatically receives a queued email containing their payslip PDF as an attachment — eliminating the manual download-and-distribute step.

**Architecture:** A new queued listener (`EmailPayslipPdfOnPayrollFinalized`) reacts to the existing `PayrollPeriodFinalized` event. For each payroll row in the period, it (a) loads the employee + email, (b) generates the payslip PDF binary via `PayslipPdfService::generate()`, (c) sends a `PayslipMail` Mailable with the binary attached, and (d) stamps `payrolls.payslip_emailed_at` so retries / re-finalize are no-ops. A new boolean setting `payroll.payslip_email.enabled` (default `true`) gates the whole behaviour. Per-employee opt-out is deferred — uses the existing `notification_preferences` table only if a `payslip_email` channel row exists; if not present, the channel is treated as opted-in.

**Tech Stack:** Laravel 11 Mail (SMTP via Mailpit in dev), Markdown Mailable, queue:default. Reuses `PayslipPdfService` (no PDF re-implementation), `SettingsService` (flag), and the existing `PayrollPeriodFinalized` event.

### Task C: Mailable — `PayslipMail`

**Files:** Create `api/app/Modules/Payroll/Mail/PayslipMail.php`

- [ ] **Step 1: Write the Mailable**

```php
<?php

declare(strict_types=1);

namespace App\Modules\Payroll\Mail;

use App\Modules\Payroll\Models\Payroll;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Mail\Mailable;
use Illuminate\Mail\Mailables\Attachment;
use Illuminate\Mail\Mailables\Content;
use Illuminate\Mail\Mailables\Envelope;
use Illuminate\Queue\SerializesModels;

class PayslipMail extends Mailable implements ShouldQueue
{
    use Queueable, SerializesModels;

    public function __construct(
        public readonly Payroll $payroll,
        private readonly string $pdfBinary,
        private readonly string $pdfFilename,
    ) {}

    public function envelope(): Envelope
    {
        $start = $this->payroll->period?->period_start?->format('Y-m-d') ?? 'period';
        return new Envelope(
            subject: "Payslip — {$start}",
        );
    }

    public function content(): Content
    {
        return new Content(
            markdown: 'emails.payslip',
            with: [
                'employee' => $this->payroll->employee,
                'period'   => $this->payroll->period,
                'payroll'  => $this->payroll,
            ],
        );
    }

    /**
     * @return array<int, Attachment>
     */
    public function attachments(): array
    {
        return [
            Attachment::fromData(fn () => $this->pdfBinary, $this->pdfFilename)
                ->withMime('application/pdf'),
        ];
    }
}
```

### Task E: Listener — `EmailPayslipPdfOnPayrollFinalized`

**Files:** Create `api/app/Modules/Payroll/Listeners/EmailPayslipPdfOnPayrollFinalized.php`

- [ ] **Step 1: Write the listener**

```php
<?php

declare(strict_types=1);

namespace App\Modules\Payroll\Listeners;

use App\Common\Services\SettingsService;
use App\Modules\Payroll\Events\PayrollPeriodFinalized;
use App\Modules\Payroll\Mail\PayslipMail;
use App\Modules\Payroll\Models\Payroll;
use App\Modules\Payroll\Services\PayslipPdfService;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Support\Facades\Log;
use Illuminate\Support\Facades\Mail;

class EmailPayslipPdfOnPayrollFinalized implements ShouldQueue
{
    public function __construct(
        private readonly PayslipPdfService $pdf,
        private readonly SettingsService $settings,
    ) {}

    public function handle(PayrollPeriodFinalized $event): void
    {
        if (! (bool) $this->settings->get('payroll.payslip_email.enabled', true)) {
            return;
        }

        $payrolls = Payroll::query()
            ->where('payroll_period_id', $event->period->id)
            ->whereNull('payslip_emailed_at')
            ->with(['employee.department', 'employee.position', 'period', 'deductionDetails'])
            ->get();

        foreach ($payrolls as $payroll) {
            try {
                $email = $payroll->employee?->email;
                if (! $email) {
                    continue;
                }

                $binary   = $this->pdf->generate($payroll);
                $filename = $this->pdf->filename($payroll);

                Mail::to($email)->queue(new PayslipMail($payroll, $binary, $filename));

                $payroll->forceFill(['payslip_emailed_at' => now()])->saveQuietly();
            } catch (\Throwable $e) {
                Log::channel('single')->warning('EmailPayslipPdfOnPayrollFinalized failed for one payroll', [
                    'payroll_id' => $payroll->id,
                    'error'      => $e->getMessage(),
                ]);
                // Swallow — one bad row should not abort the rest of the batch.
            }
        }
    }
}
```

## Acceptance Criteria

1. Calling `PayrollPeriodService::finalize($period)` causes a queued `PayslipMail` to be enqueued for every `Payroll` row in that period whose employee has an `email` and `payslip_emailed_at IS NULL`.
2. Each enqueued mail carries the PDF as an attachment and uses the employee's email as the recipient.
3. Re-firing the listener (e.g. queue retry) does not duplicate mail — `payslip_emailed_at` short-circuits the row.
4. Setting `payroll.payslip_email.enabled = false` fully disables enqueueing.
5. Existing `NotifyEmployeesOnPayrollFinalized` (in-app) still fires unchanged — the two listeners are independent.
6. New tests pass; no NEW regressions in `--filter='Payroll|Notification'`.
7. No commits made.
