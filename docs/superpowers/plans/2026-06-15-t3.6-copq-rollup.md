# T3.6 — COPQ Rollup Snapshots + Spike Alert Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist monthly Cost-of-Poor-Quality breakdowns as immutable snapshots backed by per-item product cost (not the legacy `AVG(items.standard_cost)` placeholder), expose a 12-month trend endpoint for dashboards, and fire a spike alert to QC + production when MoM total cost jumps by ≥25%.

**Architecture:** Additive — one migration adds `copq_snapshots`. `CopqService::compute()` keeps its existing array shape (so `QualityDashboardService` line 53 keeps working and `CopqServiceTest` stays green) but is rewritten internally: full-calendar-month windows route through a new `snapshot($year, $month)` method that persists a row and computes per-item cost via DB-level JOIN on `non_conformance_reports.product_id → products.standard_cost` for scrap and via `work_orders.parent_ncr_id → non_conformance_reports.product_id → products.standard_cost` for rework (rework = qty × cost × 0.30). Non-month-aligned ranges (rare; dashboard does not use them) keep the ad-hoc computation path so in-progress NCRs in the current month still surface live. A `monthlyOn(1, '02:30')` console command snaps the PRIOR calendar month. `CopqSnapshotComputed` event fires `afterCommit`; queued listener `AlertOnCopqSpike` notifies `qc_inspector` + `production_manager` roles via `NotificationService` when MoM total_cost ≥ +25%.

**Tech Stack:** Laravel 11, PostgreSQL 16, existing `HasHashId` trait, existing `NotificationService::send`, existing console-schedule pattern in `api/routes/console.php`. No new packages.

**Spec source:** task description in conversation; reuses pre-existing surface listed below.

## Watch-outs (READ BEFORE STARTING)

1. **NCR.product_id → products, NOT items.** Confirmed by reading `0091_create_non_conformance_reports_table.php` line `foreignId('product_id')->constrained('products')`. `Inspection.product_id` also targets `products`. Raw materials live in `items` but NCRs are spec'd against finished goods. **Per-item cost JOIN target = `products.standard_cost` (decimal(15,2))**, NOT `items.standard_cost`. The task brief's prose mentions both; the schema is authoritative.
2. **`compute()` signature is load-bearing.** `QualityDashboardService::executive()` calls `->copq()->compute(now()->startOfMonth(), now()->endOfMonth())` and expects the existing 4-key array (`internal_failure`, `external_failure`, `total`, `period_label`). The two `CopqServiceTest` cases assert these top-level keys and `internal_failure.scrap_units`. Refactor must preserve all three contracts.
3. **Snapshot vs ad-hoc routing.** A range is "full calendar month" only when `$from->equalTo($from->copy()->startOfMonth())` AND `$to->equalTo($from->copy()->endOfMonth())` AND `$from->year === $to->year && $from->month === $to->month`. Anything else → ad-hoc. The dashboard call satisfies the rule, BUT the dashboard wants live (in-progress) numbers for the current month, so the snapshot path must rebuild every call (call `snapshot()` which uses `updateOrCreate`) rather than read a stale row. Snapshots are still useful: they persist a row whose `breakdown` jsonb is the canonical history, and the cron pins the prior month at month rollover.
4. **`updateOrCreate` keyed on (`period_year`, `period_month`).** The migration's UNIQUE composite guards against duplicates. Re-running the cron for the same month is a no-op-with-refresh — computed numbers may differ if NCRs were closed late, which is intentional.
5. **`prevention_cost` + `appraisal_cost` stay 0.** No data sources exist for training-hours-costed or inspection-labor-costed totals yet. The columns ship now so the snapshot schema is stable, but the service writes 0.0 explicitly. Spec is explicit: keep the columns, store zero.
6. **Per-item cost JOIN must survive `null` `product_id`.** Older NCRs may have `product_id IS NULL`. INNER JOIN drops them; that is correct (no cost without product). Document this in the service so the lower number isn't mistaken for a regression. Returns/complaints stay count-based for now (no per-item join) — `external_complaint_cost` and `external_return_cost` write 0.0 alongside the count.
7. **`closed_at` window for scrap.** Scrap cost only counts NCRs where `status = 'closed'` AND `disposition = 'scrap'` AND `closed_at` falls in the window. An NCR opened in May but closed in June counts toward June. Rework window keys off `work_orders.created_at` (when rework was scheduled), not the parent NCR's `closed_at`.
8. **Decimals come back as strings.** `DB::table()->sum()` on a `decimal` column returns a string in PostgreSQL via the PHP driver. Cast through `(float)` before arithmetic, then `round(..., 2)` before persisting / returning.
9. **Event afterCommit + queued listener.** Without `afterCommit`, the event fires while the snapshot row is uncommitted, the listener queues, and the worker reads a row that may not yet exist (or worse, a stale row from a rolled-back txn). Wrap `snapshot()` in `DB::transaction()` and dispatch via `event()` inside — Laravel 11 honors `ShouldDispatchAfterCommit` on the event class. Listener implements `ShouldQueue`. Tests use `Queue::fake()` + `Event::fake()` OR set `queue.default = sync` in `setUp` — pick `Event::fake` for "event dispatched" assertions, `Queue::fake` + sync dispatch for "notification sent" assertions; do not mix.
10. **Spike threshold = +25% strictly.** Compare `new.total_cost` to prior month's persisted snapshot's `total_cost`. If prior is missing → skip (no baseline). If prior is `0.00` → skip (division by zero, treat as no baseline). If new < prior or new == prior → skip. Otherwise `(new - prior) / prior >= 0.25` → fire. Test cases: (a) prior 100, new 130 → fire, (b) prior 100, new 110 → skip, (c) prior 100, new 100 → skip, (d) prior 0, new 50 → skip, (e) no prior row → skip.
11. **Notification recipients.** `User::whereHas('role', fn ($q) => $q->whereIn('slug', ['qc_inspector', 'production_manager']))->where('is_active', true)->get()`. Mirror the pattern from `NotifyOnInspectionFailed` (api/app/Modules/Quality/Listeners/NotifyOnInspectionFailed.php). Wrap entire listener body in `try/catch (\Throwable $e)` + `Log::warning(...)` — must never crash the queue worker.
12. **Permission slug.** `quality.copq.view` does not exist in `RolePermissionSeeder.php` (confirmed by grep). Add to the quality permission block (~line 245) and grant to: `system_admin` (auto via `*`), `qc_inspector`, `production_manager`, `plant_manager`, `executive`/`vp` if present. Re-run seeder in dev to bind.
13. **Existing `CopqServiceTest` data shape.** Test creates ONE NCR with `disposition='scrap'`, `affected_quantity=10`, `closed_at = startOfMonth + 1 day`, no `product_id`. After refactor with INNER JOIN on products, scrap_units would drop to 0 for that row. **Fix in Task 3:** keep returning `scrap_units` based on COUNT/SUM of the raw NCR rows in window (no JOIN dependency), and use the JOIN result only for the `*_cost` figures. The legacy test asserts `scrap_units = 10`, not `scrap_cost`. Verify before committing.
14. **Trend endpoint pagination.** `?months=12` default. Cap server-side at 60 (5 years) to prevent abuse. Order by `(period_year, period_month) ASC` so the SPA charts left-to-right without re-sorting. Return all numeric columns + computed period label (e.g. `"2026-06"`). Return `data: [...]` envelope to match other quality endpoints.
15. **Console command idempotency.** `copq:snap-monthly` with no args snaps `now()->subMonthNoOverflow()` (prior month). Accept optional `--year=` `--month=` flags for backfill. `updateOrCreate` makes re-runs safe. Use `subMonthNoOverflow()` not `subMonth()` so March-1st run doesn't land on Feb-29 in a non-leap year.

```php
use App\Modules\Quality\Events\CopqSnapshotComputed;
use App\Modules\Quality\Models\CopqSnapshot;
use Carbon\Carbon;
use Illuminate\Support\Facades\Schema;

public function snapshot(int $year, int $month): CopqSnapshot
{
    $from = Carbon::create($year, $month, 1)->startOfDay();
    $to   = $from->copy()->endOfMonth();
    $fromTs = $from->toDateTimeString();
    $toTs   = $to->toDateTimeString();

    return \DB::transaction(function () use ($year, $month, $from, $to, $fromTs, $toTs) {

        // --- Internal failure: scrap cost (product-cost JOIN) ---
        $scrapCost = (float) (\DB::table('non_conformance_reports as n')
            ->join('products as p', 'n.product_id', '=', 'p.id')
            ->where('n.status', 'closed')
            ->where('n.disposition', 'scrap')
            ->whereBetween('n.closed_at', [$fromTs, $toTs])
            ->sum(\DB::raw('n.affected_quantity * p.standard_cost')) ?? 0);

        // Keep unit counts JOIN-free for legacy `scrap_units` semantic.
        $scrapUnits = (int) (\DB::table('non_conformance_reports')
            ->where('status', 'closed')
            ->where('disposition', 'scrap')
            ->whereBetween('closed_at', [$fromTs, $toTs])
            ->sum('affected_quantity') ?? 0);

        // --- Internal failure: rework cost (WO -> parent NCR -> product) ---
        $reworkCost = (float) (\DB::table('work_orders as w')
            ->join('non_conformance_reports as n', 'w.parent_ncr_id', '=', 'n.id')
            ->join('products as p', 'n.product_id', '=', 'p.id')
            ->whereNotNull('w.parent_ncr_id')
            ->whereBetween('w.created_at', [$fromTs, $toTs])
            ->sum(\DB::raw('w.quantity_target * p.standard_cost * 0.30')) ?? 0);

        $reworkUnits = (int) (\DB::table('work_orders')
            ->whereNotNull('parent_ncr_id')
            ->whereBetween('created_at', [$fromTs, $toTs])
            ->sum('quantity_target') ?? 0);

        // --- External failure (counts only, costs stay 0 for now) ---
        $returns = Schema::hasTable('return_requests')
            ? (int) \DB::table('return_requests')
                ->where('status', 'completed')
                ->whereBetween('updated_at', [$fromTs, $toTs])
                ->count()
            : 0;

        $complaints = Schema::hasTable('customer_complaints')
            ? (int) \DB::table('customer_complaints')
                ->whereBetween('created_at', [$fromTs, $toTs])
                ->count()
            : 0;

        $internalScrap  = round($scrapCost, 2);
        $internalRework = round($reworkCost, 2);
        $externalReturn = 0.00;       // no per-item cost yet
        $externalComp   = 0.00;       // no per-item cost yet
        $total = round($internalScrap + $internalRework + $externalReturn + $externalComp, 2);

        $breakdown = [
            'internal_failure' => [
                'scrap_units'  => $scrapUnits,
                'rework_units' => $reworkUnits,
                'scrap_cost'   => $internalScrap,
                'rework_cost'  => $internalRework,
            ],
            'external_failure' => [
                'returns'      => $returns,
                'complaints'   => $complaints,
                'return_cost'  => $externalReturn,
            ],
            'total'        => $total,
            'period_label' => $from->format('M Y'),
        ];

        $snap = CopqSnapshot::updateOrCreate(
            ['period_year' => $year, 'period_month' => $month],
            [
                'prevention_cost'         => 0.00,
                'appraisal_cost'          => 0.00,
                'internal_scrap_cost'     => $internalScrap,
                'internal_rework_cost'    => $internalRework,
                'external_return_cost'    => $externalReturn,
                'external_complaint_cost' => $externalComp,
                'total_cost'              => $total,
                'breakdown'               => $breakdown,
                'computed_at'             => now(),
            ],
        );

        event(new CopqSnapshotComputed($snap));

        return $snap;
    });
}
```

```php
public function compute(CarbonInterface $from, CarbonInterface $to): array
{
    if ($this->isFullCalendarMonth($from, $to)) {
        $snap = $this->snapshot($from->year, $from->month);

        // Legacy contract: same array shape the dashboard + tests expect.
        $b = $snap->breakdown;
        return [
            'internal_failure' => $b['internal_failure'],
            'external_failure' => $b['external_failure'],
            'total'            => (float) $b['total'],
            'period_label'     => $from->format('M Y') . ' – ' . $to->format('M Y'),
        ];
    }

    return $this->computeAdHoc($from, $to);
}

private function isFullCalendarMonth(CarbonInterface $from, CarbonInterface $to): bool
{
    return $from->year === $to->year
        && $from->month === $to->month
        && $from->equalTo($from->copy()->startOfMonth())
        && $to->equalTo($from->copy()->endOfMonth());
}

private function computeAdHoc(CarbonInterface $from, CarbonInterface $to): array
{
    // Preserved legacy ad-hoc path (avg(items.standard_cost) fallback) for
    // non-month-aligned ranges. Dashboards never hit this in practice.
    $fromDate = $from->toDateString();
    $toDate   = $to->toDateString();

    $scrap = (int) (\DB::table('non_conformance_reports')
        ->where('status', 'closed')->where('disposition', 'scrap')
        ->whereBetween('closed_at', [$fromDate, $toDate])->sum('affected_quantity') ?? 0);
    $rework = (int) (\DB::table('work_orders')
        ->whereNotNull('parent_ncr_id')
        ->whereBetween('created_at', [$fromDate, $toDate])->sum('quantity_target') ?? 0);
    $returns = (int) \DB::table('return_requests')
        ->where('status', 'completed')->whereBetween('updated_at', [$fromDate, $toDate])->count();
    $complaints = (int) \DB::table('customer_complaints')
        ->whereBetween('created_at', [$fromDate, $toDate])->count();

    $avgCost    = (float) (\DB::table('items')->avg('standard_cost') ?? 50.0);
    $scrapCost  = $scrap * $avgCost;
    $reworkCost = $rework * $avgCost * 0.3;

    return [
        'internal_failure' => [
            'scrap_units' => $scrap, 'rework_units' => $rework,
            'scrap_cost'  => round($scrapCost, 2), 'rework_cost' => round($reworkCost, 2),
        ],
        'external_failure' => [
            'returns' => $returns, 'complaints' => $complaints, 'return_cost' => 0.0,
        ],
        'total' => round($scrapCost + $reworkCost, 2),
        'period_label' => $from->format('M Y') . ' – ' . $to->format('M Y'),
    ];
}
```

```php
<?php

declare(strict_types=1);

namespace App\Modules\Quality\Listeners;

use App\Common\Services\NotificationService;
use App\Modules\Auth\Models\User;
use App\Modules\Quality\Events\CopqSnapshotComputed;
use App\Modules\Quality\Models\CopqSnapshot;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Support\Facades\Log;

class AlertOnCopqSpike implements ShouldQueue
{
    private const THRESHOLD = 0.25;

    public function __construct(private readonly NotificationService $notifications) {}

    public function handle(CopqSnapshotComputed $event): void
    {
        try {
            $snap = $event->snapshot;
            $prior = $this->priorMonth($snap);

            if (! $prior) return;
            $priorTotal = (float) $prior->total_cost;
            if ($priorTotal <= 0.0) return;

            $newTotal = (float) $snap->total_cost;
            $delta = ($newTotal - $priorTotal) / $priorTotal;
            if ($delta < self::THRESHOLD) return;

            $audience = User::whereHas('role', fn ($q) =>
                    $q->whereIn('slug', ['qc_inspector', 'production_manager'])
                )
                ->where('is_active', true)
                ->get();

            if ($audience->isEmpty()) return;

            $pct = number_format($delta * 100, 1);
            $label = sprintf('%04d-%02d', $snap->period_year, $snap->period_month);

            $this->notifications->send($audience, 'quality.copq_spike', [
                'title'       => "COPQ Spike — {$label}",
                'message'     => "COPQ jumped {$pct}% MoM to ₱"
                                . number_format($newTotal, 2) . '.',
                'link_to'     => '/quality/copq',
                'entity_type' => 'copq_snapshot',
                'entity_id'   => $snap->hash_id,
            ]);
        } catch (\Throwable $e) {
            Log::warning('AlertOnCopqSpike failed', ['error' => $e->getMessage()]);
        }
    }

    private function priorMonth(CopqSnapshot $snap): ?CopqSnapshot
    {
        $py = $snap->period_year;
        $pm = $snap->period_month - 1;
        if ($pm === 0) { $pm = 12; $py--; }

        return CopqSnapshot::where('period_year', $py)
            ->where('period_month', $pm)
            ->first();
    }
}
```

```php
declare(strict_types=1);

namespace App\Console\Commands;

class SnapCopqMonthly extends Command

    protected $description = 'Persist a COPQ snapshot row for the previous calendar month (idempotent).';

    public function handle(CopqService $copq): int
    {
        $target = ($this->option('year') && $this->option('month'))
            ? Carbon::create((int) $this->option('year'), (int) $this->option('month'), 1)
            : Carbon::now()->subMonthNoOverflow()->startOfMonth();

        $this->info("Snapped COPQ for {$target->format('Y-m')} — total ₱{$snap->total_cost}");
        return self::SUCCESS;
    }
```
