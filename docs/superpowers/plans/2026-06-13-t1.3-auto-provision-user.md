# Track 1, Task 3: Auto-Provision User Account on Employee Hire — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` syntax.

**Goal:** When an `Employee` row is created, automatically provision a system `User` account (email login, default role `employee`, temp password, force change on first login) and email a welcome message — no HR admin button-click required.

**Architecture:** A new queued listener `AutoProvisionUserOnEmployeeHire` reacts to the existing `EmployeeCreated` event. It delegates to the already-built `UserProvisioningService::provisionForEmployee()` which handles user creation, password history, role assignment, and welcome email via `WelcomeNotification`. The listener is gated by the new setting `hr.auto_provision_user.enabled` (default `true`), is idempotent (the underlying service throws `DomainException` on duplicate which the listener swallows), and follows the same try/catch + Log::warning pattern as the existing `InitializeLeaveBalances` listener so a provisioning failure cannot fail the hire flow.

**Tech Stack:** Laravel 11 events + queued notifications. Reuses `UserProvisioningService` (no copy-paste of provisioning logic). Reuses `WelcomeNotification` (no new mailable).

## Background Knowledge for the Implementer

- `EmployeeCreated` event: `api/app/Modules/HR/Events/EmployeeCreated.php`. Public `$employee` property holds the `Employee` model. Fired in `EmployeeService::create` via `DB::afterCommit(...)` so when listener handles, the row is committed.
- `UserProvisioningService::provisionForEmployee(Employee $employee, array $options = []): User` at `api/app/Modules/HR/Services/UserProvisioningService.php`:
  - Throws `\DomainException('Employee already has a system account.')` if `$employee->user()->exists()` — this is our idempotency guard.
  - Generates a temp password and email automatically when `options['email']` is missing.
  - Sends `WelcomeNotification` when `options['send_welcome']` is missing or true.
  - Default role resolved via `Role::where('slug', 'employee')` then `'self_service'` then any non-admin.
- Existing matching listener style: `api/app/Modules/HR/Listeners/InitializeLeaveBalances.php` — queued, try/catch, log on error. Mirror exactly.
- AppServiceProvider listener registrations live in `boot()` around lines 130-170. Existing line: `Event::listen(EmployeeCreated::class, [InitializeLeaveBalances::class, 'handle']);` — add ours right after.
- The Employee model exposes `email` directly (column on the table). It does NOT need a User to exist before. `UserProvisioningService` builds an email from first/last name when `email` is absent, so we don't have to short-circuit on missing email.
- Settings flag read pattern is the same as T1.1/T1.2: `(bool) $settings->get('hr.auto_provision_user.enabled', true)`.

## Tasks

### Task A: Listener — `AutoProvisionUserOnEmployeeHire`

**Files:** Create `api/app/Modules/HR/Listeners/AutoProvisionUserOnEmployeeHire.php`

- [ ] **Step 1: Write the listener**

```php
<?php

declare(strict_types=1);

namespace App\Modules\HR\Listeners;

use App\Common\Services\SettingsService;
use App\Modules\HR\Events\EmployeeCreated;
use App\Modules\HR\Services\UserProvisioningService;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Support\Facades\Log;

class AutoProvisionUserOnEmployeeHire implements ShouldQueue
{
    public function __construct(
        private readonly UserProvisioningService $provisioning,
        private readonly SettingsService $settings,
    ) {}

    public function handle(EmployeeCreated $event): void
    {
        if (! (bool) $this->settings->get('hr.auto_provision_user.enabled', true)) {
            return;
        }

        try {
            $this->provisioning->provisionForEmployee($event->employee);
        } catch (\DomainException $e) {
            // Account already exists — re-fire of the event is a no-op.
            Log::info('AutoProvisionUserOnEmployeeHire skipped: '.$e->getMessage(), [
                'employee_id' => $event->employee->id,
            ]);
        } catch (\Throwable $e) {
            // Provisioning must never fail the hire flow.
            Log::warning('AutoProvisionUserOnEmployeeHire failed', [
                'employee_id' => $event->employee->id,
                'error'       => $e->getMessage(),
            ]);
        }
    }
}
```

### Task D: Feature test

**Files:** Create `api/tests/Feature/HR/AutoProvisionUserOnHireTest.php`

- [ ] **Step 1: Write the test**

```php
<?php

declare(strict_types=1);

namespace Tests\Feature\HR;

use App\Modules\Auth\Models\Role;
use App\Modules\Auth\Models\User;
use App\Modules\Auth\Notifications\WelcomeNotification;
use App\Modules\HR\Events\EmployeeCreated;
use App\Modules\HR\Listeners\AutoProvisionUserOnEmployeeHire;
use App\Modules\HR\Models\Employee;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Illuminate\Support\Facades\Notification;
use Tests\TestCase;

class AutoProvisionUserOnHireTest extends TestCase
{
    use RefreshDatabase;

    protected function setUp(): void
    {
        parent::setUp();
        // The 'employee' role must exist for default-role resolution.
        Role::firstOrCreate(['slug' => 'employee'], ['name' => 'Employee']);
    }

    public function test_employee_created_event_provisions_user_and_sends_welcome(): void
    {
        Notification::fake();

        $employee = Employee::factory()->create(['email' => 'newhire@ogami.test']);

        app(AutoProvisionUserOnEmployeeHire::class)->handle(new EmployeeCreated($employee));

        $this->assertDatabaseHas('users', [
            'employee_id' => $employee->id,
            'is_active'   => true,
            'must_change_password' => true,
        ]);

        Notification::assertSentTo(
            User::where('employee_id', $employee->id)->first(),
            WelcomeNotification::class,
        );
    }

    public function test_listener_is_idempotent_when_user_already_exists(): void
    {
        Notification::fake();

        $employee = Employee::factory()->create();
        // Pre-existing user account.
        User::factory()->create([
            'employee_id' => $employee->id,
            'email'       => 'existing@ogami.test',
        ]);

        app(AutoProvisionUserOnEmployeeHire::class)->handle(new EmployeeCreated($employee));

        // No duplicate user, no welcome email.
        $this->assertSame(1, User::where('employee_id', $employee->id)->count());
        Notification::assertNothingSent();
    }

    public function test_feature_flag_off_disables_provisioning(): void
    {
        Notification::fake();
        app(\App\Common\Services\SettingsService::class)
            ->set('hr.auto_provision_user.enabled', false, 'hr');
        \Illuminate\Support\Facades\Cache::forget('settings:hr.auto_provision_user.enabled');

        $employee = Employee::factory()->create();

        app(AutoProvisionUserOnEmployeeHire::class)->handle(new EmployeeCreated($employee));

        $this->assertSame(0, User::where('employee_id', $employee->id)->count());
        Notification::assertNothingSent();
    }
}
```

3. With the setting `hr.auto_provision_user.enabled = false`, no user row is created.

5. New tests pass; no NEW regressions in `--filter='HR|Auth|Employee'` beyond pre-existing user-WIP failures.
