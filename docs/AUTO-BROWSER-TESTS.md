```

  SO(draft‚Üíconfirmed) ‚Üí MRP plan ‚Üí WO(planned‚Üíconfirmed‚Üíin_progress‚Üícompleted‚Üíclosed)

  ‚Üí GRN(pending_qc‚Üíaccepted) [gated: incoming inspection=passed] ‚Üí weighted-avg-cost recompute

  Hire ‚Üí Shift assign ‚Üí Biometric CSV ‚Üí DTR ‚Üí Leave(pending_dept‚Üípending_hr‚Üíapproved)

# PART 3 ‚Äî `auto-browser` Execution Scripts

## 3.0 One-time setup (per role profile)

```
# Bootstrap each role as a reusable auth profile (manual-login-once model)
for (profile, email) in the table in ¬ßEnvironment:
  browser.create_session {name: profile, start_url: "http://localhost/login"}
  browser.act "Type {email} in email, 'password' in password, click Sign in;
               wait for the dashboard to load."
  browser.save_profile {session_id, profile_name: profile}
# Portals log in at /portal/supplier/login and /portal/customer/login.
```

Each test = `OPEN[P_x]` (create_session from a saved profile, already authed) ‚Üí
`act` directives ‚Üí **ASSERT** via `GET /sessions/<id>/observe`. For unattended CI,
the terminal assertion of each test is mirrored as a convergence task-contract under
`evals/contracts/` (see ¬ß3.2).

> The MCP/REST primitives behind every `act`/`observe`:
> ```bash
> curl -s http://127.0.0.1:8000/sessions -X POST -H 'content-type: application/json' \
>   -d '{"name":"t1","start_url":"http://localhost/login"}' | jq        # -> {id}
> curl -s http://127.0.0.1:8000/mcp/tools/call -X POST -H 'content-type: application/json' \
>   -d '{"name":"browser.act","arguments":{"session_id":"<id>","instruction":"<directive>"}}'
> curl -s http://127.0.0.1:8000/sessions/<id>/observe | jq              # -> assertion surface
> ```
