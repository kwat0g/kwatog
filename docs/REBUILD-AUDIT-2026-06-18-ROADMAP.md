# Ogami ERP — 6-Month Roadmap (from 2026-06-18 audit)

> Companion to `docs/REBUILD-AUDIT-2026-06-18.md`. Sprint = 2 weeks.
> Calibrated to the post-`wave1/2/3` + `t3.x` state, where the prior audit's P0/P1 backlog
> is already shipped. This roadmap covers only the **remaining second layer** + pilot
> enablement + commercialization polish.
>
> Each sprint delivers user-visible value and is dependency-aware. REC-NN refers to the
> recommendation cards in the main report; OGAMI-NNN to the backlog tickets.

## Milestone view

| Month | Theme | Headline deliverables | Demo-able outcome |
|-------|-------|----------------------|-------------------|
| M1 | Statutory-grade correctness | REC-03 fresh gov tables, REC-02 remittance forms, REC-04 fin idempotency | "Here are this month's 1601-C, RF-1, MCRF, generated by the system" |
| M2 | JP parent reporting (the differentiator) | REC-01 multi-currency + JPY consolidation pack | "Here is the Tokyo monthly pack: TB in PHP and JPY with the FX rate" |
| M3 | Money-path hardening + demo polish | REC-06 optimistic locking, REC-07 3-way fallback, REC-08 demo bundle | Concurrency conflict shows a clean 409; voided invoice prints VOID |
| M4 | Pilot enablement | REC-05 migration toolkit (opening balances + master data) | "We loaded Ogami's real opening TB and 200 employees from Excel" |
| M5 | Differentiation depth + reporting | REC-09 JP layouts/i18n (scoped), REC-10 recurring/reversing JE + reopened-period report | JP 請求書 invoice; auditor report of reopened-period postings |
| M6 | Hardening, pilot run, defense prep | parallel-run support, lint-grade bucket-G pass, restore-drill rehearsal, defense script | Live end-to-end demo across all 3 chains + statutory + JP pack |

## Sprint detail

### Sprint 1 (M1a) — Government tables + first remittance forms
- **Objective:** every payslip computes the *current* statutory amount and the system emits
  the most-used monthly forms.
- **Deliverables:** OGAMI-101 (REC-03, gov tables 2025/2026 + effective-date test);
  OGAMI-102 (REC-02 slice: 1601-C + 2307).
- **Dependencies:** none. Do tables first (forms read them).
- **Risk:** official 2026 schedules must be sourced accurately (cite the agency circulars).
- **Demo target:** generate a 1601-C for the seeded payroll month.

### Sprint 2 (M1b) — Remaining remittance forms + financial idempotency
- **Objective:** finish the statutory form set; make money-doc creation safe under retry.
- **Deliverables:** OGAMI-103 (REC-02 slice: RF-1, MCRF, R-5, 1604-CF); OGAMI-104 (REC-04
  Idempotency-Key on Invoice/Bill/JE).
- **Dependencies:** Sprint 1 (tables).
- **Demo target:** statutory-exports screen listing all forms by period; double-submit an
  invoice → single document.

### Sprint 3–4 (M2) — Multi-currency core + JPY consolidation pack
- **Objective:** prove the JP-parent differentiator.
- **Deliverables:** OGAMI-105 (REC-01: currencies + fx_rates tables, seeded 12-mo rates);
  OGAMI-106 (ConsolidationService TB translation + CTA); OGAMI-107 (月次決算 PDF pack + export).
- **Dependencies:** existing TB/statement services.
- **Risk:** translation method correctness (current vs average rate) — write tests against a
  known worked example.
- **Demo target:** one-click "Tokyo monthly pack" with PHP + JPY columns.

### Sprint 5 (M3a) — Optimistic locking + 3-way hardening
- **Deliverables:** OGAMI-108 (REC-06 lock_version on money/payroll tables + 409 UI);
  OGAMI-109 (REC-07 require item_id / manual mapping, drop positional fallback).
- **Demo target:** two tabs editing one invoice → conflict surfaced.

### Sprint 6 (M3b) — Demo-correctness bundle
- **Deliverables:** OGAMI-110 (REC-08: payslip 503→service, status watermarks, ₱/date sweep).
- **Demo target:** clean click-through of all 3 chains with correct print artifacts.

### Sprint 7–8 (M4) — Migration toolkit
- **Deliverables:** OGAMI-111 (REC-05: per-entity importers dry-run→validate→commit→rollback);
  OGAMI-112 (opening-balance JE importer + TB-match reconciliation report).
- **Dependencies:** period-lock, sequences, JE service.
- **Demo target:** import a sample opening TB + 200-employee master file and reconcile.

### Sprint 9 (M5a) — JP layouts + scoped i18n
- **Deliverables:** OGAMI-113 (REC-09: i18next JA on finance + invoice/monthly-close PDF JP layout).
- **Demo target:** render a 請求書-style invoice.

### Sprint 10 (M5b) — GL reporting depth
- **Deliverables:** OGAMI-114 (REC-10: recurring + reversing JE templates + reopened-period report).
- **Demo target:** auditor-facing "postings to reopened periods" report.

### Sprint 11–12 (M6) — Hardening + pilot + defense
- **Objective:** pilot-credible and defense-ready.
- **Deliverables:** parallel-run support (1–2 payroll cycles + 1 month-end alongside legacy);
  lint-grade bucket-G pass (PHPStan + structured grep harness — the sweep the workflow tooling
  couldn't complete); restore-drill rehearsal against `docs/RESTORE-DRILL.md`; defense script
  walking all 3 chains + statutory + JP pack.
- **Demo target:** the full live defense run.

## Critical path
REC-03 → REC-02 (forms read tables) → independent of REC-01.
REC-01 is the longest single item (M2, ~3–4 weeks effective) — start its schema early.
REC-05 (migration) gates the *real* pilot but not the *demo*; if defense precedes pilot,
M4 can swap with M5.

## If defense is sooner than pilot
Compress to: Sprint 1 (REC-03) → Sprint 2 (REC-04 + REC-08 demo bundle) → Sprint 3–4 (REC-01
JPY pack) → Sprint for REC-02 forms. Defer REC-05 migration + REC-09 JP-i18n to post-defense,
since they prove pilot/commercial readiness rather than thesis depth.
